    学习规划

    1. C Primer Plus（第6版）Linux开发核心知识精准规划（仅实战刚需）

    ## 一、整体说明

    本章节仅整理Linux C开发、嵌入式、工程实操必须用到的知识点，剔除教材冗余理论、应试冷门知识点，完全适配8天实操学习计划，所有内容均为后续代码开发、项目实战、底层原理铺垫的核心刚需内容。

    ## 二、各章节精准刚需知识点（仅学有用内容）

    ### 第10章 指针（核心重中之重，Linux底层必备）

    必学刚需知识：

    • 基础指针原理：指针变量定义、地址取值（&）、解引用（*）、指针赋值与传参
    • 指针与数组：数组首地址、指针遍历数组、数组名与指针的区别
    • 指针数组 vs 数组指针：定义格式、本质区别、实际使用场景（高频考点+开发常用）
    • 二级指针：二级指针定义、用于修改一级指针地址、字符串数组操作
    • 字符串指针：字符串指针定义、指针操作字符串、字符串遍历、字符串工具函数实现原理

    直接跳过：指针冷门运算、老式指针兼容语法、纯理论指针推导

    ### 第12章 动态内存分配（堆内存核心）

    必学刚需知识：

    • 堆、栈、全局内存区核心区别（结合第9章存储类理解）
    • 核心函数：malloc、free 函数原型、参数、返回值、标准使用规范
    • 动态内存实操：一维动态数组、二维动态数组开辟与释放
    • 动态链表基础：节点结构体定义、节点开辟、增删改查基础逻辑
    • 内存问题排查：野指针成因与危害、内存泄漏场景、重复释放内存问题

    直接跳过：calloc、realloc 冷门用法、内存分配底层源码理论

    ### 第13章 C标准文件IO（Linux系统IO前置）

    必学刚需知识：

    • 文件指针 FILE 结构体核心作用
    • 核心IO函数：fopen、fclose、fread、fwrite、fgets、fputs 标准用法
    • 文件打开权限：r/w/a/r+/w+/a+ 权限区别与实战选用规则
    • 缓冲区原理：用户态缓冲区、内核态缓冲区区别、IO缓冲机制对读写的影响
    • 文本文件读写实操：逐行读写、批量读写、文件内容覆盖与追加

    直接跳过：格式化IO冷门参数、文件定位偏移复杂用法、临时文件操作

    ### 第14章 结构体+联合体+枚举+位段+函数指针

    前半段刚需：结构体、联合体、枚举、位段（驱动开发必备）

    • 结构体：结构体定义、结构体变量、结构体初始化、结构体成员访问
    • 结构体指针：结构体指针传参、指针访问成员（->）、嵌套结构体
    • 联合体：共用内存原理、大小端适配场景、数据类型转换实战用法
    • 枚举：枚举定义、常量封装、代码状态标识实战使用
    • 位段：位域定义、内存位操作、寄存器配置底层应用（嵌入式/Linux驱动核心）

    后半段刚需：函数指针与回调函数

    • 函数指针定义、函数地址获取、函数指针变量赋值
    • 回调函数原理：函数指针作为函数参数、回调机制实现
    • 简易回调框架封装：解耦代码逻辑、通用功能封装实战

    直接跳过：结构体内存对齐复杂理论、老式结构体兼容语法

    ### 第16章 预处理+宏+条件编译+工程规范

    必学刚需知识：

    • 宏定义：无参宏、带参宏定义与使用、宏与函数的区别
    • 条件编译：#ifdef / #ifndef / #else / #endif 核心语法与实战场景
    • 头文件保护宏：标准防重复包含宏写法（工程必备规范）
    • 多文件工程规范：.h头文件、.c源文件分工规则
    • 文件拆分原则：函数声明、宏定义、结构体定义放.h，函数实现放.c

    直接跳过：预处理冷门指令、复杂宏嵌套语法、废弃预处理规则

    ### 穿插复习 第9章 存储类&多文件基础

    必学刚需知识：

    • 四大存储类：auto、static、extern、register 核心特性（重点static、extern）
    • static作用：局部变量静态化、全局变量/函数私有化
    • extern作用：跨文件声明全局变量、声明外部函数
    • 内存分区：栈区、堆区、全局静态区、常量区内存特性
    • 多文件基础：跨文件函数调用、变量引用规则

    ## 四、针对性防遗忘实用学习建议（解决学完就忘、记不牢、不会用）

    结合本次8天高强度实操学习特点，针对绝大多数人C语言学了就忘、看懂不会写、过两天归零的通病，整理一套可直接落地的学习规则，全程适配指针、内存、结构体、工程开发学习，从根源巩固记忆。

    ### 1. 拒绝“看书看懂=学会”，坚持零抄写默写

    这是防遗忘最核心的规则。你之前遗忘的核心原因是：被动看知识点、抄例题，大脑没有形成独立编码记忆。学习所有章节（指针、动态内存、文件IO、结构体）时，严格执行：看一遍原理 → 关掉文档/书本 → 独立手写代码 → 出错再复盘。

    尤其是指针、动态内存、函数指针这类抽象知识点，只要能独立敲出来、不看参考、一次跑通，记忆留存率直接提升80%，彻底避免“看懂了，手写就废，过两天就忘”的问题。

    ### 2. 每章节强制做对比记忆，杜绝模糊混淆遗忘

    C语言最容易遗忘、混淆的全是成对知识点，模糊记忆是遗忘的根源，学习时必须当场对比总结，不堆积模糊知识点：

    • 指针章节：当场区分 指针数组/数组指针/二级指针，手写对比案例，记录一行核心区别
    • 内存章节：固定牢记 栈内存/堆内存 生命周期、申请释放规则
    • 文件IO：区分 缓冲区读写差异、不同打开权限 的使用场景
    • 工程章节：区分 static/extern、.h/.c文件分工

    每学完一个模块，不用写长篇笔记，只写3行核心区别总结，考前、复盘快速过一遍，永久不混淆、不遗忘。

    ### 3. 遵循当日学、当日复盘、次日复用记忆节奏

    遗忘的核心规律：新知识24小时内遗忘速度最快。适配你8天学习节奏，固定复盘规则：

    • 每晚结束：花15分钟，默写当天核心知识点+敲2个核心代码案例
    • 次日学习前：先用5分钟跑一遍昨日代码，不看笔记独立运行

    例如：昨天学完动态内存，今天学文件IO前，先独立手写一次malloc开辟+free释放，简单复盘即可，不用重学全套，即可牢牢锁住记忆，避免学新忘旧。

    ### 4. 刻意制造错误调试记忆，比顺写代码更难忘

    顺利跑通的代码记忆很浅，报错、排错的记忆是长期记忆。完全贴合你的学习任务，刻意实操易错点：

    • 动态内存：故意不free制造内存泄漏、故意访问野指针看报错
    • 指针：故意写错指针数组和数组指针，编译报错后修正
    • 多文件工程：故意不写头文件保护宏，制造重复包含报错并解决

    调试过一次的错误，终身难忘，彻底解决“学完只会写标准答案，换场景就废”的问题。

    ### 5. 所有知识点绑定场景记忆，拒绝纯理论背诵

    纯文字知识点最容易忘，绑定Linux/工程开发场景，记忆会形成条件反射：

    • 学指针：绑定“字符串处理、数组遍历、函数传参”场景
    • 学动态内存：绑定“链表、动态数组、工程可变内存”场景
    • 学位段/结构体：绑定“Linux驱动、寄存器配置、数据封装”场景
    • 学函数指针：绑定“回调解耦、通用函数框架”场景
    • 学预处理/多文件：绑定“大型项目工程规范、跨文件开发”场景

    每学一个知识点，清楚它在哪用、用来解决什么问题，就不会快速遗忘。

    ### 6. 禁止“断层式学习”，保持知识点串联记忆

    你本次的学习顺序10→12→13→14→16是连贯的，严禁跳学、断更。C语言知识点是链式关联：指针支撑动态内存、动态内存支撑结构体链表、函数指针支撑工程框架、预处理支撑多文件开发。

    连贯学习会形成知识网络，单个知识点遗忘时，整体网络会帮你推导回忆，彻底告别零散、学一个忘一个的情况。

    ### 7. 最终防遗忘杀手锏：8天结束综合Demo闭环

    最后一天的综合刷题+多文件Demo是巩固记忆的关键，不要敷衍。将所有知识点串联进一个项目：指针操作+动态内存链表+结构体+文件IO存储+函数指针回调+多文件拆分+头文件保护全部整合。

    1数组

    c中的数组需要初始化，否则会存储垃圾值，如下

    #include <stdio.h>
#include <stdlib.h>
#define MAX 4
int main()
{
    int a[MAX];//未初始化
    for (int i = 0; i < MAX; i++)
    {
        printf("a[%d] = %d\n", i, a[i]);
    }
    return 0;
}

    两次结果运行发现，未初始化的数组中的元素会出现未知的数值。

    对于初始化后，或者只初始化部分值，其他未赋值的元素会被初始化为0

    #define MAX 4
int main()
{
    int a[MAX]={12};
    for (int i = 0; i < MAX; i++)
    {
        printf("a[%d] = %d\n", i, a[i]);
    }
    return 0;
}

    使用const关键字修饰时，运行时无法修改数组中的值

    2指针

    第一部分：基础一级指针（判断输出 + 画图）

    题 1

    #include <stdio.h>
int main() {
    int a = 20;
    int *p = &a;
    *p = 66;
    printf("a=%d\n", a);
    printf("*p=%d\n", *p);
    return 0;
}

    问题：输出结果是多少？画图说明内存变化。

    a=20 //a=99

    *p=66

    说明：*p=66 解引用指针 p，直接修改 a 内存里的数据。

    题 2

    #include <stdio.h>
int main() {
    int x = 1, y = 99;
    int *ptr = &x;
    ptr = &y;
    printf("%d", *ptr);
    return 0;
}

    问题：输出多少？ptr = &y 这一行做了什么（修改值还是修改指向）？

    输出：99

    修改了指针的指向，由原来的x地址改成指向y

    第二部分：函数传一级指针（修改普通变量）

    题 3

    #include <stdio.h>
void change(int *num) {
    *num = 1000;
}
int main() {
    int a = 10;
    change(&a);
    printf("%d", a);
    return 0;
}

    问题：输出结果？如果函数参数改成 int num，调用 change(a)，结果会变成多少？为什么？

    输出：1000

    10

    因为形参为指针时，修改的是该值的地址所在的内容，而当形参为一个普通变量时，传入的是一个值的副本，不会影响该值。

    第三部分：二级指针核心题（你最疑惑的修改指针指向）

    题 4（对标你原来的代码）

    #include <stdio.h>
void set(int **pp) {
    int temp = 888;
    *pp = &temp;
}
int main() {
    int a = 10;
    int *p = &a;
    set(&p);
    printf("%d", *p);
    return 0;
}

    问题：

    1. set(&p) 为什么不能写成 set(p)？
    2. 函数里为什么写 *pp = &temp，不是 **pp = &temp？
    3. 这段代码存在什么安全隐患？

    说明：

    pp 是int**二级指针，需要传入指针变量 p 的地址&p；p 是int*类型，类型不匹配；

    *pp等价于 main 里的 p，给 p 赋值新地址；**pp等价于 a，是 int 数字，不能存地址；

    temp 是函数局部变量，函数结束后销毁，p 变成悬垂野指针，输出随机垃圾值。

    题 5

    #include <stdio.h>
void func(int **pp) {
    **pp = 500;
}
int main() {
    int a = 10;
    int *p = &a;
    func(&p);
    printf("a=%d", a);
    return 0;
}

    问题：输出多少？**pp = 500 修改的是谁，和上一题 *pp = 地址 有什么本质区别？

    修改的是a

    说明：输出 a=500；

    **pp=500：修改 p 指向的整型数据 a；

    *pp=地址：修改指针 p 本身的指向，两个操作对象完全不同。
